## 🎯 Tổng Quan Dự Án

**Calorie Web** là ứng dụng nhẹ để theo dõi calo với các mục tiêu:

- ✅ **Đơn giản** - Giao diện dễ sử dụng
- ✅ **Nhanh** - Hiệu suất tối ưu
- ✅ **Type-safe** - Full TypeScript support
- ✅ **Hiện đại** - Next.js & React mới nhất

**Trạng thái:** v0.1.0 (Early Stage Development)  
**Đối tượng sử dụng:** Sinh viên, người tập gym, người quan tâm sức khỏe

## 🔗 Cấu Trúc Dữ Liệu

### Hệ Thống Kiểu (Type System)

Tất cả interface được định nghĩa trong `types/index.ts`:

```
MacroTarget (Mục tiêu dinh dưỡng)
    ├── calories: number
    ├── protein: number
    ├── carbs: number
    └── fat: number
         ↓ (contained in)

UserProfile (Thông tin người dùng)
    ├── name: string
    ├── age: number
    ├── weight: number
    ├── height: number
    ├── goal: 'lose' | 'maintain' | 'gain'
    └── macroTarget: MacroTarget

Ingredient (Nguyên liệu)
    ├── name: string
    ├── calories: number
    ├── protein: number
    ├── carbs: number
    ├── fat: number
    └── amount: number
         ↓ (multiple in)

MealEntry (Một bữa ăn)
    ├── id: string
    ├── mealType: 'breakfast' | 'lunch' | 'dinner' | 'snack'
    ├── ingredients: Ingredient[]
    ├── totalCalories: number
    └── time: string (HH:mm)
         ↓ (multiple per day)

DailyLog (Nhật ký hàng ngày)
    ├── date: string (YYYY-MM-DD)
    ├── meals: MealEntry[]
    └── totalCalories: number
```

### Bảng Quan Hệ Dữ Liệu

| Interface       | Mục Đích             | Chứa         | Được Chứa Trong | Số Lượng |
| --------------- | -------------------- | ------------ | --------------- | -------- |
| **MacroTarget** | Mục tiêu dinh dưỡng  | (không)      | UserProfile     | 1:1      |
| **UserProfile** | Thông tin người dùng | MacroTarget  | Zustand Store   | 1:1      |
| **Ingredient**  | Dữ liệu thực phẩm    | (không)      | MealEntry       | 1:N      |
| **MealEntry**   | Một bữa ăn           | Ingredient[] | DailyLog        | 1:N      |
| **DailyLog**    | Dữ liệu ngày         | MealEntry[]  | localStorage    | 1:N      |

## 💾 Storage Layer

### Chiến Lược Lưu Trữ

Module `lib/storage.ts` cung cấp abstraction layer an toàn cho localStorage:

```typescript
// Đọc dữ liệu
const profile = getProfile(); // UserProfile | null
const dailyLog = getLog("2024-05-08"); // DailyLog | null
const weekLogs = getLogs("2024-05-01", "2024-05-08"); // DailyLog[]

// Lưu dữ liệu
saveProfile(userProfile);
saveLog("2024-05-08", dailyLog);
```

### Khóa Storage

```
calorie_profile          → UserProfile
calorie_log_2024-05-08   → DailyLog cho ngày cụ thể
calorie_log_2024-05-07   → DailyLog cho ngày khác
...
```

### Lợi Ích

- ✅ Safe JSON parsing (không crash nếu dữ liệu hỏng)
- ✅ Browser-safe (kiểm tra window/localStorage existence)
- ✅ Centralized (một chỗ quản lý storage logic)
- ✅ Type-safe (trả về typed objects)
- ✅ Dễ migrate (có thể swap sang backend API sau)

## 📦 State Management với Zustand

### Cấu Hình Store Hiện Tại

```typescript
import { create } from "zustand";

type AppState = {
  test: string;
  setTest: (value: string) => void;
};

export const useAppStore = create<AppState>((set) => ({
  test: "",
  setTest: (value) => set({ test: value }),
}));
```

### Tại Sao Zustand?

| Tiêu Chí    | Zustand   | Redux | Context     |
| ----------- | --------- | ----- | ----------- |
| Bundle Size | ~1KB      | ~40KB | ~0KB        |
| Học Tập     | Rất dễ    | Khó   | Trung bình  |
| Boilerplate | Tối thiểu | Nhiều | Vừa         |
| Hiệu Năng   | Tuyệt vời | Tốt   | Có thể chậm |

### Cấu Trúc Store Khuyến Nghị (Giai Đoạn Tiếp)

```typescript
type AppState = {
  // User Profile
  userProfile: UserProfile | null;
  setUserProfile: (profile: UserProfile) => void;

  // Daily Logs
  dailyLogs: Map<string, DailyLog>;
  addMeal: (date: string, meal: MealEntry) => void;

  // UI State
  isLoading: boolean;
  error: string | null;
};
```

## 🛠️ Quy Trình Phát Triển

### Standard Development Process

```
1. Tạo Branch
   └─ feature/* hoặc bugfix/* từ main

2. Phát Triển
   └─ Code theo conventions
   └─ Test ở development server

3. Kiểm Tra Chất Lượng
   └─ npm run lint
   └─ Sửa lỗi ESLint

4. Commit & Push
   └─ Conventional commits
   └─ git push origin branch-name

5. Pull Request
   └─ Tạo PR trên GitHub
   └─ Request review

6. Merge
   └─ Sau khi approved
   └─ Xóa feature branch
```

### Commit Convention

```

fix: Sửa lỗi
docs: Cập nhật tài liệu
refactor: Tái cấu trúc code
```

## 📝 Code Standards

### TypeScript

- Luôn define return types
- Dùng interfaces cho object types
- Strict mode bắt buộc

### Components

- Functional components + hooks
- PascalCase filenames (Button.tsx)
- Một component per file
- JSDoc comments cho public APIs

### State Management

- Zustand cho global state
- Keep slices focused
- TypeScript cho store types

### Styling

- Tailwind CSS utilities
- Không inline styles
- CSS variables cho custom values
