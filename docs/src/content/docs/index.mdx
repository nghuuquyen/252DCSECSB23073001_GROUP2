## 🎯 Tổng Quan Dự Án

**Calorie Web** là ứng dụng web hiện đại giúp người dùng:
- Ghi nhật ký calo từng ngày với 4 bữa ăn (Breakfast, Lunch, Dinner, Snack)
- Thêm/xóa nguyên liệu cho từng bữa, nhập số gram, tự động tính calories, protein, carbs, fat
- Tìm kiếm nhanh món ăn Việt Nam phổ biến (15+ món)
- Tính tổng calories từng bữa và cả ngày, hiển thị rõ ràng
- Lưu trữ tự động qua localStorage (Zustand), không mất dữ liệu khi reload
- Giao diện mobile-first, tối ưu cho sinh viên/người mới bắt đầu

**Trạng thái:** v0.1.0 (Early Stage Development)
**Đối tượng sử dụng:** Sinh viên, người tập gym, người quan tâm sức khỏe

### useProfileStore

**Tập tin:** `store/profileStore.ts`

**Mục đích:** Quản lý thông tin profile người dùng toàn cục

**State & Actions:**
- `profile: UserProfile | null` - Thông tin người dùng hiện tại
- `setProfile(profile)` - Thiết lập profile mới hoàn toàn
- `updateProfile(partial)` - Cập nhật một phần thông tin profile
- `loadProfile()` - Tải profile từ localStorage khi ứng dụng khởi động
- `syncStreak()` - Đồng bộ chuỗi ngày streak từ logs, lưu vào localStorage và cập nhật Zustand store

**Integration:**
- Tự động persist sang localStorage (thông qua helper `lib/storage.ts`)
- Được sử dụng rộng rãi trong `app/settings/page.tsx`, `components/dashboard/DashboardPage.tsx`, `app/stats/page.tsx` để duy trì dữ liệu nhất quán.

### useDiaryStore

**Tập tin:** `store/diaryStore.ts`

**Mục đích:** Quản lý nhật ký calo hàng ngày

**State:**
- `currentLog: DailyLog | null` - Nhật ký hiện tại
- `currentDate: string` - Ngày đang xem (YYYY-MM-DD)
- `loadLog(date)` - Load nhật ký cho ngày cụ thể
- `addMeal(meal)` - Thêm bữa ăn mới
- `removeMeal(mealId)` - Xóa bữa ăn
- `updateMeal(mealId, updated)` - Cập nhật bữa ăn
- `updateWater(water)` - Cập nhật lượng nước

**Features:**
- Tự động tính `totalCalories` từ tất cả meals
- Persist tự động sang localStorage
- Support multiple meals per day

✅ **Diary UI**: Giao diện nhập nhật ký calo, 4 section bữa ăn, modal thêm nguyên liệu, xóa nguyên liệu, tổng calories từng bữa và cả ngày.

✅ **Hồ sơ người dùng & Settings**: Thiết lập hồ sơ người dùng, chọn giới tính, tuổi, cân nặng, chiều cao, mức độ vận động và mục tiêu sức khỏe (giảm/duy trì/tăng cân).
✅ **Tính toán dinh dưỡng**: Tính toán BMR, TDEE và mục tiêu macros (Protein, Carbs, Fat) tự động theo công thức Mifflin-St Jeor khoa học.

✅ **Lưu trữ tự động**: Cơ chế Zustand + localStorage giúp ứng dụng hoạt động hoàn toàn offline, giữ nguyên dữ liệu khi reload.
✅ **Responsive**: Giao diện mobile-first, tối ưu mượt mà cho điện thoại và máy tính bảng.

## 🔗 Cấu Trúc Dữ Liệu

### Hệ Thống Kiểu (Type System)

Tất cả interface được định nghĩa trong `types/index.ts`:

```
MacroTarget (Mục tiêu dinh dưỡng)
    ├── calories: number
    ├── protein: number
    ├── carbs: number
    └── fat: number
         ↓ (contained in)

UserProfile (Thông tin người dùng)
    ├── name: string
    ├── age: number
    ├── weight: number
    ├── height: number
    ├── goal: 'lose' | 'maintain' | 'gain'
    └── macroTarget: MacroTarget

Ingredient (Nguyên liệu)
    ├── name: string
    ├── calories: number
    ├── protein: number
    ├── carbs: number
    ├── fat: number
    └── amount: number
         ↓ (multiple in)

MealEntry (Một bữa ăn)
    ├── id: string
    ├── mealType: 'breakfast' | 'lunch' | 'dinner' | 'snack'
    ├── ingredients: Ingredient[]
    ├── totalCalories: number
    └── time: string (HH:mm)
         ↓ (multiple per day)

DailyLog (Nhật ký hàng ngày)
    ├── date: string (YYYY-MM-DD)
    ├── meals: MealEntry[]
    └── totalCalories: number
```

### Bảng Quan Hệ Dữ Liệu

| Interface       | Mục Đích             | Chứa         | Được Chứa Trong | Số Lượng |
| --------------- | -------------------- | ------------ | --------------- | -------- |
| **MacroTarget** | Mục tiêu dinh dưỡng  | (không)      | UserProfile     | 1:1      |
| **UserProfile** | Thông tin người dùng | MacroTarget  | Zustand Store   | 1:1      |
| **Ingredient**  | Dữ liệu thực phẩm    | (không)      | MealEntry       | 1:N      |
| **MealEntry**   | Một bữa ăn           | Ingredient[] | DailyLog        | 1:N      |
| **DailyLog**    | Dữ liệu ngày         | MealEntry[]  | localStorage    | 1:N      |

## 🏗️ Kiến Trúc & Luồng Dữ Liệu

### Luồng Dữ Liệu Ứng Dụng

```
┌──────────────────────────────────┐
│   User Interaction               │
│   (Forms, Buttons, Input)        │
└──────────────┬───────────────────┘
               │
       ┌───────▼────────┐
       │  Zustand Store │
       │  useAppStore   │
       │                │
       │  - Profile     │
       │  - Logs        │
       │  - Settings    │
       └───────┬────────┘
               │
    ┌──────────┴──────────┐
    │                     │
┌───▼────────┐      ┌────▼─────┐
│localStorage│      │React State│
│(Persistence)      │(Rendering)
└───────┬────────┘      └────┬─────┘
        │                     │
        └──────────┬──────────┘
                   │
         ┌─────────▼────────┐
         │ UI Components    │
         │(Tailwind Styles) │
         └──────────────────┘
```

### Quy Trình Làm Việc Điển Hình

```
1. NGƯỜI DÙNG MỞ ỨNG DỤNG
   └─ Load dữ liệu từ localStorage
   └─ Populate Zustand store

2. NGƯỜI DÙNG ĐIỀU HƯỚNG
   └─ Click navigation (Next.js routing)
   └─ Load page component

3. NGƯỜI DÙNG NHẬP DỮ LIỆU
   └─ Điền form (meal, ingredients...)
   └─ Component update local state

4. NGƯỜI DÙNG GỬI
   └─ Validate dữ liệu
   └─ Update Zustand store
   └─ Persist vào localStorage
   └─ UI components re-render

5. NGƯỜI DÙNG XEM PHÂN TÍCH
   └─ Charts fetch dữ liệu từ Zustand
   └─ Recharts render visualization

6. SESSION KẾT THÚC
   └─ Dữ liệu vẫn lưu trong localStorage
```

## 💾 Storage Layer

### Chiến Lược Lưu Trữ

Module `lib/storage.ts` cung cấp abstraction layer an toàn cho localStorage:

```typescript
// Đọc dữ liệu
const profile = getProfile(); // UserProfile | null
const dailyLog = getLog("2024-05-08"); // DailyLog | null
const weekLogs = getLogs("2024-05-01", "2024-05-08"); // DailyLog[]

// Lưu dữ liệu
saveProfile(userProfile);
saveLog("2024-05-08", dailyLog);
```

### Khóa Storage

```
calorie_profile          → UserProfile
calorie_log_2024-05-08   → DailyLog cho ngày cụ thể
calorie_log_2024-05-07   → DailyLog cho ngày khác
...
```

### Lợi Ích

- ✅ Safe JSON parsing (không crash nếu dữ liệu hỏng)
- ✅ Browser-safe (kiểm tra window/localStorage existence)
- ✅ Centralized (một chỗ quản lý storage logic)
- ✅ Type-safe (trả về typed objects)
- ✅ Dễ migrate (có thể swap sang backend API sau)

## 📦 State Management với Zustand

### Cấu Hình Store Hiện Tại

```typescript
import { create } from "zustand";

type AppState = {
  test: string;
  setTest: (value: string) => void;
};

export const useAppStore = create<AppState>((set) => ({
  test: "",
  setTest: (value) => set({ test: value }),
}));
```

### Tại Sao Zustand?

| Tiêu Chí    | Zustand   | Redux | Context     |
| ----------- | --------- | ----- | ----------- |
| Bundle Size | ~1KB      | ~40KB | ~0KB        |
| Học Tập     | Rất dễ    | Khó   | Trung bình  |
| Boilerplate | Tối thiểu | Nhiều | Vừa         |
| Hiệu Năng   | Tuyệt vời | Tốt   | Có thể chậm |

### Cấu Trúc Store Khuyến Nghị (Giai Đoạn Tiếp)

```typescript
type AppState = {
  // User Profile
  userProfile: UserProfile | null;
  setUserProfile: (profile: UserProfile) => void;

  // Daily Logs
  dailyLogs: Map<string, DailyLog>;
  addMeal: (date: string, meal: MealEntry) => void;

  // UI State
  isLoading: boolean;
  error: string | null;
};
```

## 🚀 Bắt Đầu Nhanh

### Yêu Cầu

- Node.js 18+ (khuyến nghị 20+)
- npm 9+ hoặc yarn 3+
- Git

### Cài Đặt

1. **Clone dự án**

   ```bash
   git clone <repository-url>
   cd calorie-web
   ```

2. **Cài đặt dependencies**

   ```bash
   npm install
   ```

3. **Chạy development server**

   ```bash
   npm run dev
   ```

4. **Mở browser**

| **start** | `next start`              | Run production build     |

## 📝 Code Standards

### TypeScript

- Luôn define return types
- Dùng interfaces cho object types
- Strict mode bắt buộc
- Import types với `import type`

### Components

- Functional components + hooks
- PascalCase filenames (Button.tsx)
- Một component per file
- JSDoc comments cho public APIs

### State Management

- Zustand cho global state
- Keep slices focused
- TypeScript cho store types

### Styling

- Tailwind CSS utilities
- Không inline styles
- CSS variables cho custom values
